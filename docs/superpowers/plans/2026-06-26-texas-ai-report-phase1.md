# Texas AI Report — Phase 1 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Launch texasaireport.com as an Astro static news publication — Google-News-eligible, in the Momentum/navy-amber brand — on the existing `EWRDigital-com/texasaireport.com` repo → Vercel pipeline, with 8–12 seed articles, then cut the domain over.

**Architecture:** Static site generated by Astro (no server). Articles are Markdown files in a content collection; templates render the page + `NewsArticle` JSON-LD + feeds. Push to `main` → Vercel builds (`astro build`) → live + sitemaps/RSS regenerate. Human PR review is the editorial gate (pipeline automation is Phase 2, out of scope here).

**Tech Stack:** Astro 5 (static output), `@astrojs/sitemap`, `@astrojs/rss`, `@astrojs/mdx`, Vitest (unit tests for pure logic), Playwright (render smoke tests), Vercel (hosting), GitHub (`EWRDigital-com/texasaireport.com`, public).

## Global Constraints

- **Brand tokens (exact):** ink `#0C0A09`; brand navy `#0A2A47`; accent amber `#F59E0B`; dark amber (small text on white) `#B45309`; paper `#ffffff`. Amber only for accents/fills/CTAs; navy for body text links.
- **Repo MUST stay public** — required for free Vercel Hobby git-integration on an org repo (a private org repo returns 409 → Pro). See `internals-systems/reference_static_site_github_vercel_pipeline`.
- **Deploys originate from GitHub only.** Never `vercel deploy` from local; pushes to `main` auto-deploy.
- **Project name has no dots:** Vercel project is `texasaireport-com`; the domain `texasaireport.com` is attached separately.
- **Every article:** visible byline "By Matt Bertram" linking to matthewbertram.com; dateline + published/updated timestamps; `NewsArticle` schema with `author` = Matt (`sameAs` → matthewbertram.com, modalpoint.com); a "Why it matters" key-points box.
- **ModalPoint links:** only where the compliance angle is genuinely relevant, AND always with the disclosure line + the Ownership & Funding Disclosure page present.
- **No intrusive interstitials/popups** (Google News/Discover penalize them).
- **YMYL accuracy:** verify statute facts against primary sources before publishing; every article carries an "analysis, not legal advice" footer line.
- **Google News requirements baked in:** original human-edited content, bylines, datelines, `NewsArticle` + `NewsMediaOrganization` schema, Google News sitemap (last 48h), standard sitemap, RSS, trust pages, consistent name/logo, fast + mobile + HTTPS + stable URLs.

```ts
import { SITE, MATT } from '../config/site';

const person = {
  '@type': 'Person', name: MATT.name, url: MATT.url, sameAs: MATT.sameAs,
  jobTitle: MATT.role, description: MATT.bioShort,
};
export function orgSchema() {
  return { '@context':'https://schema.org', '@type':'NewsMediaOrganization',
    name: SITE.name, url: SITE.url, logo: SITE.logo, founder: person, slogan: SITE.tagline };
}
export function articleSchema(a: {
  title:string; dek:string; url:string; publishDate:Date; updatedDate?:Date;
  image?:string; section:string;
}) {
  return { '@context':'https://schema.org', '@type':'NewsArticle',
    headline: a.title, description: a.dek, url: a.url,
    mainEntityOfPage: a.url, articleSection: a.section,
    image: a.image ? [a.image] : undefined,
    datePublished: a.publishDate.toISOString(),
    dateModified: (a.updatedDate ?? a.publishDate).toISOString(),
    author: person,
    publisher: { '@type':'NewsMediaOrganization', name: SITE.name, logo: { '@type':'ImageObject', url: SITE.logo } } };
}
```

export function recentItems<T extends { publishDate: Date }>(items: T[], now = new Date()): T[] {
  const cutoff = now.getTime() - 48 * 3600 * 1000;
  return items.filter(i => i.publishDate.getTime() >= cutoff);
}

```ts
import type { APIRoute } from 'astro';
import { getCollection } from 'astro:content';
import { recentItems } from '../lib/news-sitemap';
import { SITE } from '../config/site';
const esc=(s:string)=>s.replace(/&/g,'&amp;').replace(/</g,'&lt;');
export const GET: APIRoute = async () => {
  const all=(await getCollection('news')).filter(e=>!e.data.draft);
  const recent=recentItems(all.map(e=>({ id:e.id, data:e.data, publishDate:e.data.publishDate })));
  const urls=recent.map(e=>`<url><loc>${SITE.url}/news/${e.id}/</loc>`
    +`<news:news><news:publication><news:name>${esc(SITE.name)}</news:name>`
    +`<news:language>en</news:language></news:publication>`
    +`<news:publication_date>${e.data.publishDate.toISOString()}</news:publication_date>`
    +`<news:title>${esc(e.data.title)}</news:title></news:news></url>`).join('');
  const xml=`<?xml version="1.0" encoding="UTF-8"?>`
    +`<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9" `
    +`xmlns:news="http://www.google.com/schemas/sitemap-news/0.9">${urls}</urlset>`;
  return new Response(xml,{headers:{'Content-Type':'application/xml'}});
};
```
